## ⚙️ New Configuration System

### **Settings Added (Properties/Settings.settings):**

| Setting | Type | Default | Purpose |
|---------|------|---------|---------|
| `DeploymentLogDirectory` | string | *Database path + "\DeploymentLogs"* | PowerShell script logs |
| `WindowsUpdateISOPath` | string | *(empty)* | Windows Update ISO file path |
| `LocalISOHostnamePattern` | string | `"TN"` | Hostname pattern for ISO deployment |
| `LocalISOHostnameMatchMode` | string | `"StartsWith"` | Pattern matching mode |

### **Options Menu UI (OptionsWindow.xaml):**

**Location:** Options → Database & Configuration → Deployment Configuration

#### **1. Deployment Log Directory**
- Text box with folder browser button (📁)
- **Default:** `{DatabasePath}\DeploymentLogs`
- **Purpose:** Where PowerShell scripts save update logs
- **Automatic fallback:** `%TEMP%\NecessaryAdminTool_Logs` if path unavailable

#### **2. Windows Update ISO Path**
- Text box with file browser button (📁)
- **Default:** *(empty - use cloud Windows Update)*
- **Purpose:** Path to Windows Feature Update ISO file
- **Format:** Network UNC path or local path (e.g., `\\server\share\Win11.iso`)

#### **3. Local ISO Deployment Pattern**
- Dropdown: Starts With / Equals / Contains / Ends With
- Text box for pattern (e.g., "TN")
- **Default:** "Starts With" + "TN"
- **Purpose:** Which computers use ISO vs cloud update
- **Example:** Computers starting with "TN" use local ISO, others use cloud

## 📖 User Documentation

### **For Administrators:**

#### **To Configure Deployment Paths:**
1. Open NecessaryAdminTool
2. Go to **Options** (⚙️ button)
3. Scroll to **Deployment Configuration** section
4. Configure:
   - **Log Directory:** Where scripts save logs (defaults to database location)
   - **ISO Path:** Path to Windows Update ISO (optional)
   - **Hostname Pattern:** Which computers use ISO vs cloud
5. Click **SAVE AND CLOSE**

#### **To Use PowerShell Scripts:**
**Option A: Download and Run Manually**
1. Deployment Center → Download Scripts
2. Deploy to endpoints via RMM/GPO
3. Scripts will read paths from environment variables OR use local fallback

**Option B: Set Environment Variables in RMM**
```powershell
# ManageEngine / RMM Custom Fields
[Environment]::SetEnvironmentVariable("NECESSARYADMINTOOL_LOG_DIR", "\\server\share\Logs", "Machine")
[Environment]::SetEnvironmentVariable("NECESSARYADMINTOOL_ISO_PATH", "\\server\share\Win11.iso", "Machine")
```
